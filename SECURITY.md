# Security Policy

## Supported Versions

Security fixes are provided for the latest public release on PyPI.

If a vulnerability affects older versions, upgrade to the latest release first.

## Reporting a Vulnerability

Please do not open public GitHub issues for security vulnerabilities.

Use one of the following private channels:

1. GitHub security advisories:
   - Go to the Security tab in this repository.
   - Use Report a vulnerability.
2. Email:
   - support@qmill.com

Include the following details when possible:

- Affected package version
- Reproduction steps or proof of concept
- Impact assessment
- Suggested mitigation

## Response Process

After receiving a report, maintainers will:

1. Acknowledge receipt
2. Validate and triage severity
3. Prepare and release a fix
4. Coordinate disclosure timing with the reporter

Thank you for helping keep QAS SDK users safe.
